# .env File Configuration Guide

This guide explains what each environment variable means and how to set it up.

## 📝 Template

Create a file named `.env` in the `backend/` folder with this content:

```env
# Database Configuration
DATABASE_URL=postgresql://username:password@host:5432/database_name

# JWT Secret (generate a random string)
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production

# Server Configuration
PORT=3000
NODE_ENV=development

# CORS (your Flutter app URL)
ALLOWED_ORIGINS=http://localhost:*,https://yourdomain.com
```

---

## 🔧 Variable Explanations

### 1. DATABASE_URL

**What it is**: PostgreSQL database connection string

**Format**: `postgresql://username:password@host:port/database_name`

#### For Local Development:
```env
DATABASE_URL=postgresql://postgres:yourpassword@localhost:5432/senyamatika
```

- `postgres` = default PostgreSQL username
- `yourpassword` = your PostgreSQL password
- `localhost` = your computer
- `5432` = default PostgreSQL port
- `senyamatika` = database name

#### For Render (Production):
```env
DATABASE_URL=postgresql://senyamatika_user:abc123xyz...@dpg-xxxxx.oregon-postgres.render.com/senyamatika_db
```

**Where to get it**:
- **Render**: Copy the "Internal Database URL" from your PostgreSQL service
- **Local**: Use your local PostgreSQL credentials

---

### 2. JWT_SECRET

**What it is**: Secret key used to sign authentication tokens

**Requirements**:
- At least 32 characters long
- Random and unpredictable
- NEVER share or commit to Git

#### How to Generate:

**Option 1: Online Generator**
- Go to: https://randomkeygen.com/
- Copy a "CodeIgniter Encryption Key" (256-bit)

**Option 2: Node.js**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

**Option 3: PowerShell**
```powershell
-join ((48..57) + (65..90) + (97..122) | Get-Random -Count 32 | % {[char]$_})
```

#### Example:
```env
JWT_SECRET=a8f5f167f44f4964e6c998dee827110c3b7c0a9a2e5f8d3c1b4a7e9f2d6c8b5a
```

⚠️ **IMPORTANT**: Use a different secret for development and production!

---

### 3. PORT

**What it is**: Port number where your API server will run

**Default**: `3000`

#### For Local Development:
```env
PORT=3000
```

#### For Render (Production):
```env
PORT=3000
```

**Note**: Render automatically assigns a port, but setting this ensures consistency.

---

### 4. NODE_ENV

**What it is**: Environment mode (development or production)

#### For Local Development:
```env
NODE_ENV=development
```

#### For Render (Production):
```env
NODE_ENV=production
```

**What it affects**:
- Logging verbosity
- Error messages (detailed in dev, generic in prod)
- Database SSL settings
- Performance optimizations

---

### 5. ALLOWED_ORIGINS

**What it is**: Domains allowed to access your API (CORS)

**Format**: Comma-separated list of URLs

#### For Local Development:
```env
ALLOWED_ORIGINS=http://localhost:*,http://127.0.0.1:*
```

This allows:
- `http://localhost:3000`
- `http://localhost:8080`
- Any localhost port

#### For Production:
```env
ALLOWED_ORIGINS=https://yourdomain.com,https://www.yourdomain.com
```

#### For Both (Recommended):
```env
ALLOWED_ORIGINS=http://localhost:*,https://yourdomain.com,https://www.yourdomain.com
```

#### Allow All (Not Recommended for Production):
```env
ALLOWED_ORIGINS=*
```

## 🚀 Quick Setup Steps

### Step 1: Copy Template
```bash
cd backend
cp .env.example .env
```

### Step 2: Edit .env File

**Windows**:
```bash
notepad .env
```

**Mac/Linux**:
```bash
nano .env
```

### Step 3: Fill in Values

1. **DATABASE_URL**: 
   - Local: Use your PostgreSQL credentials
   - Render: Copy from Render dashboard

2. **JWT_SECRET**: 
   - Generate a random 32+ character string
   - Use different secrets for dev and prod

3. **PORT**: 
   - Keep as `3000`

4. **NODE_ENV**: 
   - `development` for local
   - `production` for Render

5. **ALLOWED_ORIGINS**: 
   - Local: `http://localhost:*`
   - Prod: Your actual domain

### Step 4: Save and Test

```bash
# Test if .env is loaded correctly
node -e "require('dotenv').config(); console.log('DATABASE_URL:', process.env.DATABASE_URL ? 'Set ✅' : 'Missing ❌')"
```

## 🔒 Security Best Practices

### ✅ DO:
- Use strong, random JWT_SECRET
- Keep .env file in .gitignore
- Use different secrets for dev/prod
- Rotate secrets periodically
- Use environment variables in Render (don't commit .env)

### ❌ DON'T:
- Commit .env to Git
- Share .env file publicly
- Use simple/guessable secrets
- Use same secret across environments
- Hardcode secrets in code

## 🐛 Troubleshooting

### Issue: "Cannot connect to database"

**Check**:
1. DATABASE_URL format is correct
2. PostgreSQL is running (local)
3. Credentials are correct
4. Database exists

**Test connection**:
```bash
# Install pg client
npm install -g pg

# Test connection
psql "postgresql://username:password@host:5432/database"
```

### Issue: "JWT_SECRET is not defined"

**Check**:
1. .env file exists in backend folder
2. JWT_SECRET line has no spaces around `=`
3. Value is not empty

**Test**:
```bash
node -e "require('dotenv').config(); console.log(process.env.JWT_SECRET)"
```

### Issue: "CORS error"

**Check**:
1. ALLOWED_ORIGINS includes your Flutter app URL
2. No trailing slashes in URLs
3. Protocol (http/https) matches

- [ ] ALLOWED_ORIGINS includes your app URL
